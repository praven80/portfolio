The Architecture

DevGenius was built on a serverless AWS architecture leveraging advanced AI capabilities:

Key Architecture Highlights

  • Global Content Delivery: CloudFront with Lambda@Edge authentication for secure, low-latency access worldwide
  • Containerized Application: ECS Fargate hosts Streamlit application with auto-scaling based on demand
  • AI Orchestration: Bedrock Agent (Claude Sonnet) handles conversational flow with custom discovery questions
  • Knowledge Base RAG: OpenSearch Serverless provides context from AWS documentation and whitepapers via web crawler
  • Multi-Artifact Generation: Claude Sonnet generates diagrams, CDK code, CloudFormation templates, and documentation with streaming
  • Image Analysis: Claude Sonnet vision capabilities analyze uploaded whiteboard drawings and existing diagrams
  • Persistent Storage: DynamoDB stores conversation history and S3 stores generated artifacts for audit and retrieval

Technology Stack

Amazon CloudFront AWS Lambda@Edge Application Load Balancer Amazon ECS Fargate Streamlit (Python) Amazon Bedrock Agent Claude Sonnet Amazon OpenSearch Serverless Amazon S3 Amazon DynamoDB AWS CDK (TypeScript) Amazon VPC Amazon CloudWatch

Key Architectural Decisions

1. Conversational Architecture Building

  • Bedrock Agent: Claude Sonnet handles natural language understanding with custom orchestration prompts
  • Discovery Flow: Agent asks 3-5 targeted questions about requirements before generating solutions
  • Knowledge Base RAG: OpenSearch Serverless provides context from AWS documentation and whitepapers
  • Context Management: DynamoDB stores conversation history for iterative refinement

2. Image-to-Architecture Pipeline

  • Vision AI: Claude Sonnet analyzes uploaded images to extract components and relationships
  • Image Preprocessing: Automatic resizing/compression for files exceeding 5MB
  • Component Recognition: Identifies AWS services, connections, and data flows from diagrams
  • Conversational Refinement: Users review and adjust AI interpretation through chat

3. Multi-Artifact Generation

  • Architecture Diagrams: Claude Sonnet creates draw.io XML diagrams with AWS architecture icons
  • Dual IaC Templates: Generates both AWS CDK (TypeScript) and CloudFormation (YAML)
  • Cost Estimation: AI analyzes architecture and calculates monthly costs using AWS pricing data
  • Technical Documentation: Comprehensive markdown documentation with deployment guides
  • Streaming Generation: Real-time token streaming with continuation prompts for long outputs

4. Security & Authentication

  • Lambda@Edge: Handles authentication using cognito-at-edge library
  • Cognito: Email-based authentication with self-signup and password policies
  • Network Security: ALB accepts traffic only from CloudFront, ECS in private subnets
  • IAM Least Privilege: Separate roles for ECS, Lambda, and Bedrock

5. Serverless Infrastructure

  • ECS Fargate: Containerized Streamlit application with auto-scaling based on CPU utilization
  • CloudFront + ALB: Global content delivery with application load balancing
  • S3 + DynamoDB: Artifact storage and conversation history with encryption
  • Custom Resources: Lambda functions provision services not yet supported by CDK

Key Design Decisions

Claude Sonnet: Selected for both conversational flow (Bedrock Agent) and artifact generation (code, diagrams, docs) to balance cost, speed, and quality, providing superior reasoning for complex code generation.

ECS Fargate vs Lambda: Selected Fargate for Streamlit hosting due to stateful nature, WebSocket requirements, longer AI generation times, and better container environment control, accepting higher baseline cost for improved user experience.

Web Crawler Knowledge Base: Automatically ingests latest AWS documentation without manual updates, trading longer initial ingestion time for always-current information and reduced maintenance overhead.

Streaming vs Batch: Chose streaming responses for better UX showing real-time generation progress, implementing continuation prompts to handle max token limits while maintaining context.

Draw.io vs Proprietary Format: Selected draw.io XML for wide adoption, editability in familiar tools, and browser rendering without proprietary viewers, enabling users to further refine generated diagrams.

üí°Key Learnings & Best Practices

Technical Insights

  • Multimodal AI Power: Vision capabilities analyze images without separate OCR services. single API call handles both text and visual understanding, dramatically simplifying the architecture
  • Streaming UX Impact: Real-time token streaming improves perceived performance by 10x. users see progress immediately rather than waiting minutes for complete responses
  • Knowledge Base RAG: Vector search with OpenSearch Serverless provides more accurate, up-to-date AWS recommendations than relying solely on model training data
  • Custom Resource Patterns: Lambda functions enable provisioning services not yet supported by CDK (web crawler data sources, OpenSearch Serverless indices with specific configurations)
  • IAM Eventual Consistency: Retry logic with exponential backoff (3 attempts, 3-12 second delays) handles permission propagation delays when creating OpenSearch indices

Product & UX Insights

  • Progressive Disclosure: Two-tab interface separates conversational design from image upload. don't overwhelm users with all options upfront, let them choose their preferred workflow
  • User Control Over AI: Checkboxes for artifact generation give users control over what they need, reducing unnecessary API calls and costs while respecting user agency
  • Retry Buttons: Each artifact includes retry button for regeneration. acknowledges AI non-determinism and gives users multiple attempts to get desired results
  • Visual + Conversational: Combining diagram visualization with conversational refinement provides the best of both worlds. users can see and discuss architecture simultaneously
  • Feedback Loop: Thumbs up/down with mandatory explanations enables continuous improvement and identifies problematic patterns for model fine-tuning
  • Download All: Single button creates ZIP with all artifacts. users don't need to save each artifact individually, reducing friction in workflow
  • Deployment Safety: CloudFormation launch links open in AWS Console (not automated deployment) so users can review parameters, costs, and resources before clicking "Create Stack"
